# Share-Card Polish — Design Spec

**Author:** Jake Hoffman
**Date:** 2026-05-12
**Status:** Approved, proceeding to plan + implementation

## Why

The site is going to be shared. When the URL gets pasted into iMessage,
Slack, LinkedIn, Discord, or Twitter, the preview card is currently a
text-only stub: no image, no favicon, generic browser-chrome tinting.
This cycle adds the polish layer that makes the link feel like a
designed thing the moment anyone sees it — *before* they click.

## Goal

1. A custom 1200×630 OG image that matches v2's engineering-log
   aesthetic (mono, dark, amber accent) and remains legible at the
   ~300 px width that mobile link-preview cards render at.
2. A favicon that survives both light- and dark-mode browser chrome
   without re-uploading variants — SVG primary, ICO fallback.
3. An Apple touch icon for iOS "Add to Home Screen."
4. A complete `<meta>` block in `<head>` that every modern link-preview
   scraper recognizes (Facebook/LinkedIn/iMessage via OpenGraph,
   Twitter/X via Twitter cards, browsers via `theme-color`, OS icon
   handlers via `<link rel="icon">` family).

## Non-Goals

- Self-hosting JetBrains Mono (separate cycle, lower priority).
- Flipping the repo public / enabling GitHub Pages (your call, separate
  cycle).
- Multiple OG image variants per section / per linked URL.
- Animated OG (no platform renders anything but PNG/JPEG reliably).
- PWA manifest / installable web-app setup. Overkill for a static
  resume page.
- Custom 404 page.

**Typography:** JetBrains Mono (Bold for the name, SemiBold for the
prompt and metrics prefix, Regular for body). Falls back to whatever
mono is bundled with the rendering machine if the font isn't on disk.

**Colors:** identical to v2 dark tokens:
- background `#0a0e14`
- body `#d6dee6`
- amber accent `#f5b342`
- muted `#5a6573` (for the URL footer and metrics prefix)
- status dot color `#f5b342`

**Rendering:** a Python script at
`C:/Users/14jak/GitHub/website/scripts/render-og-image.py`, kept in the
repo for reproducibility. Uses PIL (already on the machine — used for
the [removed] WebP conversion). The script finds a JetBrains Mono TTF
on disk; if not found, it falls back to the first available mono font
on the system (Cascadia Mono, Consolas, etc.) — visually close enough
because the OG image is consumed standalone.

**Font discovery:** the script searches a small list of likely
locations in order and uses the first hit:
1. `C:/Users/14jak/AppData/Local/Microsoft/Windows/Fonts/JetBrainsMono-*.ttf`
2. `C:/Windows/Fonts/JetBrainsMono-*.ttf`
3. `C:/Windows/Fonts/CascadiaMono.ttf` / `CascadiaCode.ttf` (Windows 11 default)
4. `C:/Windows/Fonts/consola.ttf` (Consolas fallback)
If none are found, PIL's default font is used and the script prints a
warning. The OG image renders either way.

### 2. Favicon

**Primary:** `C:/Users/14jak/GitHub/website/favicon.svg`, hand-written
SVG, viewBox 0 0 32 32. A bold mono `JH` glyph centered in a rounded-
square `#0a0e14` background with a `#f5b342` accent rim. The glyph
uses `currentColor` so it auto-tints when the browser's UI is in dark
or light mode (Firefox and Safari honor this; Chrome takes the
literal fill).

**Fallback:** `C:/Users/14jak/GitHub/website/favicon.ico`, generated by
rasterizing `favicon.svg` to a 32×32 PNG via PIL and saving as ICO.
Older clients (some email apps, old Windows taskbar pinning) only
honor `.ico`.

### 3. Apple touch icon

**File:** `C:/Users/14jak/GitHub/website/apple-touch-icon.png`,
180×180 PNG, ≤ 10 KB.

Rasterized from the same SVG glyph but composed onto a 180×180 canvas
with the flat `#0a0e14` background fully filling the frame. iOS
applies its own rounded-corner mask when the user pins to home
screen — we don't pre-round.

Rendered by the same script as the OG image
(`scripts/render-og-image.py`) for one-script reproducibility.

## Verification

- Run the script; confirm all three assets render.
- Open the OG image in a viewer at 300 px width — the headline and the
  prompt line must remain legible (smallest mobile preview size).
- Validate the page in a test preview tool (e.g., [`metatags.io`](https://metatags.io/) or `opengraph.xyz`)
  after deploying. Pre-deployment, manually verify the markup with
  `curl http://localhost:8000/ | grep og:`.
- Open the favicon at the browser tab; confirm the `JH` glyph is
  recognizable at 16×16.
- Force a dark/light browser chrome (DevTools → Rendering → emulate
  `prefers-color-scheme`) and confirm `theme-color` tints correctly.

## Risks

- **PIL doesn't anti-alias SVG natively.** We never render SVG with
  PIL; the favicon SVG is shipped as-is, and the ICO + touch icon are
  rendered directly in PIL using primitives (rounded rect + text). No
  SVG-to-PNG conversion needed.
- **`og:image` requires absolute URL.** We use the GitHub Pages URL.
  Until the site is actually deployed at that URL, scrapers won't
  fetch the image — which is fine, because no one's sharing the link
  yet either.
- **`twitter:card="summary_large_image"` requires `twitter:image`.**
  Both included.
- **Theme color on iOS Safari only applies to PWAs.** Acceptable —
  this is mostly for Android Chrome and Edge anyway.
- **Browser SVG favicon rendering quirks.** Chrome won't honor
  `currentColor` for the address-bar tab indicator color; it'll just
  use the SVG's literal fill. We set the SVG glyph color to amber
  directly (good in both themes), and only the *background plate*
  uses `currentColor` so the glyph stays visible on either chrome.

## Out of Scope (YAGNI restatement)

- Multiple favicon size variants beyond what we ship (16/32/180).
- A `browserconfig.xml` for Microsoft Edge tiles.
- A site manifest (`manifest.webmanifest`) — would also unlock
  installability, but the resume isn't a web app.
- Per-section OG images (the same one suffices for every deep link).

## Success Criteria

After deployment:
- A test post on a free preview-card playground (metatags.io) shows
  the custom 1200×630 image instead of nothing.
- iMessage / Slack / LinkedIn / X all show the same image when the
  URL is pasted.
- The browser tab favicon is the amber `JH` glyph on dark, recognizable
  at 16×16.
- iOS "Add to Home Screen" places the apple-touch-icon, not a generic
  screenshot.
- Browser chrome (Chrome Mobile, Safari Mobile) tints to `#0a0e14`
  on dark and `#f5efe4` on light.
